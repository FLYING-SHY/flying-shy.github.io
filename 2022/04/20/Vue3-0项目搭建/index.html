vue3.0项目的搭建

[官网地址]: https://cli.vuejs.org/zh/ “官网地址”

  1. 在要创建项目的文件夹下边同时按shift键与鼠标右键，选择在此处打开powershell窗口

  2. 卸载之前的vue版本

  3. 全局安装脚手架

  4. 查看版本号（大写V）

  5. 在命令行安装pnpm@6的版本

  6. 搭建一个项目名称为project的项目

  7. 根据项目提示进行相应的配置，选择最后一项为手动配置

    Default ([Vue 3] babel, eslint)：默认预设配置，会快速创建一个Vue3.0项目，提供了babel和eslint支持

    Default ([Vue 2] babel, eslint)：默认预设配置，会快速创建一个Vue2.0项目，提供了babel和eslint支持

    Manually select features：手动配置项目，可根据项目的需要选择合适的选项，具备更多的选择性

  8. 移动上下箭头选择要配置的东西，按空格键表示选中（括号内会出现星标）

    Babel：使用Babel将源码进行转码（把ES6=>ES5）
    TypeScript：使用TypeScript进行源码编写。使用TypeScript可以编写强类型JavaScript
    Progressive Web App (PWA) Support：使用渐进式Web应用程序
    Router：使用Vue路由
    Vuex：使用Vuex状态管理器
    CSS Pre-processors：CSS 预处理器（如：less、sass）
    Linter / Formatter：使用代码风格检查和格式化（如：ESlint）
    Unit Testing：使用单元测试（unit tests）
    E2E Testing：使用E2E（end to end）黑盒测试

  9. 全部选好之后按回车键开始进行相应的配置

    • 选择Vue的版本为3.0

    • 选择路由采用历史模式

    • 选择css预处理器的语言：主要为css解决浏览器兼容、简化CSS代码等

    • 选择Eslint代码规范（javascript代码检测工具）

      TSLint: ts格式检验⼯具
      ESLint with error prevention only: ESLint 只会进⾏错误提醒
      ESLint + Airbnb config: ESLint Airbnb标准
      ESLint + Standard config: ESLint Standard 标准
      ESLint + Prettier: ESLint（代码质量检测）+ Prettier（代码格式化⼯具）

    • 选择何时检测

      代码检查⽅式: 保存时检查 / 提交时检查; 我一般习惯保存时检查

    • 选择单元测试的解决方案

    • 选择配置文件存放的位置

    • 是否保存当前配置作为下次创建项目的模板（我选择y，选择n则不用起名）

  10. 选择配置文件存放的位置（我选择单独保存在各自的配置文件中）

  11. 是否保存当前配置作为下次创建项目的模板（我选择y，选择n则不用起名）

  12. 搭建完成，根据提示进入文件夹

  13. 启动项目
